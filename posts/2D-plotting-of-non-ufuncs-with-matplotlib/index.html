Usable formula for long term average spectrum of music

Franz Heuchel

2017-10-25 19:52

Source

Knowing the average power spectrum of music can be useful when simulating the sound field and sound pressure levels at music events. In a recently published conference paper this quantity was computed over 12345 (what a number) popular music tracks. However, the piece-wise quadratic formula that fits this spectrum is defined over some arbitrary bins and not the frequency domain. Let's do that here.

First things first:

In [1]:
import numpy as np
import matplotlib.pyplot as plt

The authors defined their fitting function over frequency bins, labeled from 1 to 543.

In [2]:
x = np.arange(1,544)

They used two quadratic functions defined from bins 1-100 and 100-543. Also, there must be a minus sign missing with $p_1$ for $y_1$.

In [3]:
p11 = -0.000907
p12 = 0.256
p13 = -32.942
y1 = lambda x: p11*x**2 + p12*x + p13

p21 = -0.000183
p22 = 0.0213
p23 = -16.735
y2 = lambda x: p21*x**2 + p22*x + p23

plt.figure()
plt.plot(x[:99], y1(x[:99]))
plt.plot(x[99:], y2(x[99:]))
plt.grid(True)
plt.show()

The linear bin scale is actually a logarithmic frequency scale ranging from 30 Hz to 15.7 kHz. So there must be some transformation such that \begin{align*} u \log(30)+v & = 1 \\ u \log(15700)+v & = 543. \end{align*} Let's solve the linear system to find $u$ and $v$

In [4]:
A = np.array([[np.log10(30),   1],
              [np.log10(15.7e3), 1]])
b = np.array([[1],
              [543]])
u, v = np.linalg.solve(A, b)

# transformations between bin and frequency dimension
xf = lambda f: u*np.log10(f)+v
fx = lambda x: 10**((x-v)/u)

and test if we did a good job

In [5]:
f = np.logspace(np.log10(30), np.log10(15.7e3), 543, base=10)
np.allclose(x, xf(f)), np.allclose(f, fx(x))
Out[5]:
(True, True)

The frequency that splits the two quadratic functions is

In [6]:
f_switch = fx(100)
print(f_switch)

[94.12926965]

I like the spectrum to be 0 dB at its peak value, which is

In [7]:
y1(100)
Out[7]:
-16.412

So in the end we have this nice little function that takes as input a ndarray of frequencies and outputs a weighting according to the long term average of lots popular music.

In [8]:
def lta_music_weighting(f):
    peak = -16.412  # == y1(100) == y2(100)
    p11 = -0.000907
    p12 = 0.256
    p13 = -32.942 - peak
    p21 = -0.000183
    p22 = 0.0213
    p23 = -16.735 - peak
    u = 199.35423956
    v = -293.47038447
    f_switch = 94.12926965
    
    x1 = u*np.log10(f[f <= f_switch])+v
    x2 = u*np.log10(f[f >= f_switch])+v
    y1 = p11*x1**2 + p12*x1 + p13
    y2 = p21*x2**2 + p22*x2 + p23
    
    y = np.zeros(f.shape)
    y[f <= f_switch] = y1
    y[f >= f_switch] = y2
    
    return y


plt.semilogx(f, lta_music_weighting(f))
plt.grid(True)
plt.ylabel('Spectrum [dB]')
plt.xlabel('Frequency [Hz]')
plt.show()

Sweet.
